Day 03 - Access Control

What is Access Control?

Access Control is the process of deciding who is allowed to access a system, network, application, or data. It ensures that only authorized users can view or modify sensitive information.

---

Why is Access Control Important?

- Protects sensitive information
- Prevents unauthorized access
- Reduces insider threats
- Helps maintain data privacy
- Supports business security

---

Types of Access Control

1. Physical Access Control

Controls access to physical locations such as offices, server rooms, or data centers.

Examples:

- Security guards
- ID cards
- Fingerprint scanners
- Facial recognition
- Smart door locks

---

2. Logical Access Control

Controls access to digital systems and information.

Examples:

- Username and password
- PIN codes
- Multi-Factor Authentication (MFA)
- Biometric login
- Role-Based Access Control (RBAC)

---

Access Control Principles

Authentication

Verifies a user's identity.

Example:
Logging in with a username and password.

Authorization

Determines what an authenticated user is allowed to do.

Example:
An administrator can edit files, while a regular user can only view them.

Accountability

Tracks user activities through logs and monitoring.

Example:
Recording login times and file access.

---

Best Practices

- Use strong passwords.
- Enable Multi-Factor Authentication.
- Follow the Principle of Least Privilege.
- Regularly review user permissions.
- Remove access for inactive users.

---

Key Takeaways

- Access Control protects systems from unauthorized users.
- Authentication verifies identity.
- Authorization grants permissions.
- Accountability records user actions.

---

Practical Exercise

Imagine you own a small company.

List:

- Three people who should have access to the office.
- Three people who should have access to the company database.
- One person who should have administrator privileges.
